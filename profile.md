## Academic Working Preferences

- Coding style conventions: language-native formatter, no manual style debates
  (Python: Black + Ruff / PEP 8; JS/TS: Prettier + ESLint; Go: gofmt).
  Naming: descriptive, no abbreviations; snake_case (Python), camelCase (JS/TS),
  PascalCase for types/classes.
- Patterns I prefer / avoid: prefer pure functions, early returns, explicit
  over implicit, composition over inheritance. Avoid deep nesting, global
  mutable state, premature abstraction, clever one-liners.
- Testing approach: pytest (Python) / Vitest or Jest (JS/TS). Test behavior,
  not implementation. Aim for meaningful coverage of core logic, not a % target.
- Dependency philosophy: use mature, well-maintained libraries for non-trivial
  problems; avoid micro-dependencies and unmaintained packages. Pin versions,
  audit regularly.
- Citation / referencing style for written work: APA 7th for general/academic;
  IEEE for engineering/CS papers. Pick one and stay consistent.
- Comment & documentation language: English (widest collaboration reach).
  Comment the "why", not the "what".
